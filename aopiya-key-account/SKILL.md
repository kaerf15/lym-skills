# AOPIYA 大客户识别与标准化 Skill

本 skill 提供 4 个核心业务能力：

1. 客户重要性标记
2. 标准化画像
3. 大客户准入评分
4. 客户画像与行动建议

## 说明

本 skill 只负责 AOPIYA 口径下的业务判断、评分、画像整理和行动建议。

如果输入事实不足以支撑判断，应明确指出信息缺口与待确认项，不在本 skill 内扩展公开信息搜索、正文提取或补证流程。

## 何时使用

当任务涉及以下内容时，使用本 skill：

- 对海关数据、名单、访谈记录或表格中的客户做业务判断
- 判断某客户是否符合 AOPIYA 大客户准入要求
- 把零散客户信息整理成统一客户档案
- 为客户打标签，形成标准化画像
- 结合目标客户现有供应商或竞对供应商信息，说明采购背景、合作格局和替换机会
- 根据客户条件输出优先级、风险等级和跟进建议

## 模块选择

根据任务目标选择模块：

### 模块 A：客户重要性标记
功能：按 AOPIYA 口径做客户重要性标记。
执行时读取 [references/retrieval-segmentation-rules.md](references/retrieval-segmentation-rules.md)。

### 模块 B：标准化画像
功能：把客户信息整理成统一画像底稿。
执行时读取 [references/customer-information-schema.md](references/customer-information-schema.md)。

### 模块 C：准入评分
功能：对客户做量化评分和准入判断。
执行时读取 [references/admission-scoring-rubric.md](references/admission-scoring-rubric.md)。

### 模块 D：客户画像与行动建议
功能：输出画像标签、风险点和下一步动作建议。
执行时读取 [references/profile-tagging-and-actions.md](references/profile-tagging-and-actions.md)。

## 输出

优先使用 [references/output-templates.md](references/output-templates.md) 中最贴近任务的模板。

输出必须满足：

- 结构清晰，便于复制进 CRM、知识库或客户档案
- 每个判断都有依据
- 区分“事实”“判断”“建议”三类内容
- 如果包含供应商信息，需单列“供应商信息说明”或并入画像摘要，明确哪些是客户事实、哪些是供应商侧信息
- 联络信息应单列展示，并明确是否具备直接触达条件

## 禁止事项

- 不要把“男装”误判为核心高匹配行业
- 不要让加分项把单维度分数冲破上限
- 不要因为总分高就忽略账期、合规市场、竞品代理等风险
- 不要把未验证信息写成确定事实
- 不要仅凭客户现有供应商名称，就直接认定客户一定可替换、一定排他或一定有稳定大单

## 快速判断口径

如果用户只问“值不值得跟”，可以按以下最简口径快速回答：

- 优先推进：总分高、现金付款、行业匹配、门店或渠道基础好、决策人经验成熟，且具备明确联络路径
- 补证推进：基础条件较好，但采购额、付款方式、合作承诺或联络信息存在关键缺口
- 培育观察：有一定潜力，但当前规模、渠道或决策条件尚不达标
- 低优先级跟进：行业偏差大、规模过小、付款条件差、合作意愿弱，或长期无法建立有效联络路径
- 不建议准入：命中红线或核心指标明显不达标

## 建议搭配的数据源

本 skill 适合处理以下输入：

- 海关进出口客户名单
- 客户拜访纪要
- 展会收集线索
- Excel 客户表
- 经销商报名资料
- 邮件往来和官网公开信息摘要
- 目标客户现有供应商名单、竞对合作信息、主要采购品牌信息

如果输入是批量表格，优先做模块 A，再按需要进入模块 B、C、D。
