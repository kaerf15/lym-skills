# 客户标准化画像字段

本文件用于把零散客户资料统一映射为 AOPIYA 的客户档案字段，并形成可复审的标准化画像底稿。

## 一、基础信息

### 1. 基本信息

| 标准字段 | 含义 | 典型来源 | 标准化要求 |
|---|---|---|---|
| 客户编号 | 客户唯一标识 | CRM、Excel、手工编号 | 没有编号时标记为待生成 |
| 公司名称 | 客户公司全称 | 官网、名片、合同 | 保留中英文全称 |
| 国家/地区 | 客户所在国家或区域 | 官网、海关数据 | 统一用国家或地区标准名 |
| 客户类型 | 连锁零售、国际品牌商、跨境电商、区域分销商 | 官网、访谈 | 必须归入现有类型之一 |
| 主营产品 | 男包、女包、配饰、其他 | 官网、目录、访谈 | 尽量细化到实际销售品类 |
| 客户等级 | A、B、C | 内部评定 | 优先使用内部现有等级；若无内部等级，可按准入评分结果映射建议等级 |

### 2. 联络信息

联络信息属于高优先级核心字段，应优先于一般业务描述补齐。

| 标准字段 | 含义 | 标准化要求 |
|---|---|---|
| 公司地址 | 公司注册地址或办公地址 | 尽量完整保留街道、城市、邮编 |
| 公司官网 | 官方网站 | 保留主域名 |
| 联系人 | 主要对接人 | 姓名加职位 |
| 联系方式 | 电话、邮箱、社媒 | 以可联系渠道为主 |
| 联络信息完整度 | 联络路径是否足够支持销售跟进 | 标记为完整、部分缺失、严重缺失 |

### 3. 决策人信息

| 标准字段 | 含义 | 标准化要求 |
|---|---|---|
| 决策人姓名/职位 | 实际决策者身份 | 明确是否为老板、采购总监、渠道负责人 |
| 行业经验年限 | 时尚行业相关年限 | 优先记录可验证年数 |
| 合作意愿强度 | 高、中、低 | 根据明确表态，不要猜测 |
| 决策效率 | 从接触到下单的周期 | 尽量量化到月 |
| 风险偏好 | 是否接受创新合作及费用共担 | 用高、中、低或是/否归一 |
| 价值观匹配 | 是否认同环保、可持续等理念 | 需有沟通依据 |
| 社交兴趣 | 节日礼品、活动偏好 | 可用于关系维护 |
| 个人习惯 | 联系禁忌、时间偏好等 | 对跟进节奏有直接影响 |

### 4. 文化习俗

| 标准字段 | 含义 | 标准化要求 |
|---|---|---|
| 文化敏感性 | 材质、设计、图案禁忌 | 直接关联选品与沟通 |

## 二、业务信息

### 1. 企业规模

| 标准字段 | 含义 | 标准化要求 |
|---|---|---|
| 员工数 | 企业及分销团队规模 | 估算值需标注“约” |
| 年营收规模 | 公司年收入规模 | 保留币种 |
| 年采购额 | 与箱包皮具相关采购额 | 尽量转成统一币种备注 |
| 市场覆盖 | 覆盖城市、门店、分销数量 | 门店与分销数量拆开更好 |

### 2. 合作能力与资源

| 标准字段 | 含义 | 标准化要求 |
|---|---|---|
| 品牌认同 | 对 AOPIYA 品牌理念的接受度 | 用事实支撑，如是否认同环保、经典风格 |
| 成功经验 | 国际时尚品牌代理经验 | 记录品牌名、年限、结果 |
| 当前供应商情况 | 当前主要合作供应商、竞对品牌或采购来源 | 记录供应商名称、合作品类、合作状态，未知时不要猜测 |
| 竞对合作关系 | 与竞对品牌或竞对供应商的合作深度 | 区分代理、经销、采购、试单、历史合作等状态 |
| 资金状况 | 预算、流动资金、付款习惯 | 优先写付款方式和单次订单能力 |
| 服务能力 | 渠道、物流、售后、ERP 能力 | 能量化就量化 |

### 3. 业务痛点与需求

| 标准字段 | 含义 | 标准化要求 |
|---|---|---|
| 当前痛点 | 供应链、产品、营销等问题 | 用原话压缩成要点 |
| 核心需求 | 希望 AOPIYA 提供的价值 | 要和可合作方向相关 |

### 4. 购买习惯

| 标准字段 | 含义 | 标准化要求 |
|---|---|---|
| 偏好习惯 | 采购节奏、验厂、审批等 | 尽量写成流程化信息 |
| 期望价值 | 对合作成果的预期 | 优先提销量、代理权、独家款等 |

### 5. 合作状态

| 标准字段 | 含义 | 标准化要求 |
|---|---|---|
| 当前合作阶段 | 潜在客户、洽谈中、试单期、稳定合作、流失 | 必须从标准状态中选 |
| 最新跟进日期 | 最近沟通时间与摘要 | 日期加一句摘要 |
| 重点风险提示 | 红线问题或重大障碍 | 必须单列，不和一般风险混写 |

## 三、标准化输出规则

标准化画像用于把已有客户资料整理成统一字段和可复审底稿。

### 1. 字段映射原则

- 原始资料有多个同义字段时，统一映射到标准字段
- 没有直接对应字段时，保留在“补充说明”中，不要塞入相近字段
- 一个字段涉及多个事实时，拆分成短句或分号分隔

### 2. 信息可信度标记

必要时给字段加可信度：

- 高：合同、官网、财报、客户明确书面说明
- 中：销售访谈、会议纪要、展会沟通
- 低：公开二手资料、未验证转述

### 3. 缺失项提示

以下字段缺失时，应优先提醒补充：

- 联系人
- 联系方式
- 公司官网
- 公司地址
- 主营产品
- 年采购额
- 市场覆盖
- 付款方式
- 决策人经验
- 合作阶段

### 4. 标准化画像摘要口径

完成字段映射后，建议补一段简短摘要，至少包含：

- 联络信息是否完整、是否可直接触达
- 客户当前所属客群或重要性标记
- 主营品类与渠道特征
- 规模与付款条件概况
- 当前供应商格局或竞对绑定情况
- 当前最主要的机会点和缺失项

### 4.1 联络信息优先规则

- 如联系人、电话、邮箱、网址、地址明显缺失，应先补联络路径，再推进销售动作建议
- 对批量客户，可先根据联络信息完整度分层：已可触达、待补联络、暂不可触达
- 如只有官网或地址，没有联系人和直接联系方式，应标记为“部分缺失”

### 5. 供应商信息使用规则

- 供应商信息可用于说明客户当前采购结构、品牌偏好和替换难度
- 如果供应商信息来自用户、合同、官网或客户明确表述，可按事实归档
- 如果只是市场传闻或间接推测，必须标记为“待核验”
- 不要把“已有竞对供应商”直接写成“无法合作”，应结合合作深度、排他性和切入窗口判断

## 四、客户等级字段处理

客户等级字段主要用于承接企业内部已有档案，不是标准化画像阶段必须新增的判断结果。

字段处理原则：

- 优先保留企业内部已有客户等级
- 如果原始资料没有内部等级，标准化画像阶段可以暂时留空
- 如需补充“客户等级建议”，按评分规则映射补充
- 准入结论与客户等级的映射口径统一以 [admission-scoring-rubric.md](admission-scoring-rubric.md) 为准
