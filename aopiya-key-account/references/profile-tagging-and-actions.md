# 客户画像标签与行动建议

本文件定义客户画像的标准标签和对应动作。

如联络信息严重缺失，默认不进入“优先推进”，应优先归入“补证推进”或“培育观察”。

## 一、画像标签体系

为客户画像时，优先从以下维度打标签。

### 1. 业务匹配标签

- 核心品类匹配
- 价格带匹配
- 品牌调性匹配
- 渠道结构匹配

### 2. 经营能力标签

- 高采购潜力
- 强渠道覆盖
- 现金流稳健
- 服务体系完善
- ERP 协同潜力高

### 3. 决策人标签

- 行业老兵
- 资源整合型
- 长期合作型
- 数据驱动型
- 决策高效型

### 4. 风险标签

- 账期敏感
- 竞品冲突
- 供应商绑定较强
- 替换窗口待验证
- 合规待核验
- 决策链冗长
- 投入承诺不足
- 信息缺失较多

## 二、画像输出规则

画像输出建议包含以下四段：

1. 客户概况
2. 核心标签
3. 机会点
4. 风险点

如已知以下信息，建议在客户概况中同时写明：

- 重要性标记
- 准入结论
- 当前优先级
- 当前主要供应商或竞对合作情况

如果用户提供了供应商信息，画像中建议补充一段“供应商背景说明”，至少说明：

- 主要供应商是谁
- 合作品类或合作方式
- 是否存在竞对绑定
- 对切入策略的影响

## 三、行动建议分层

### 1. 优先推进

适用条件：

- 联络信息已可直接触达
- 总分高
- 现金付款
- 行业与渠道高度匹配
- 决策人经验成熟且合作意愿强

建议动作：

- 安排重点商务拜访
- 推进打样、报价和年度框架沟通
- 尽快明确首单和保底采购条件
- 如存在竞对供应商，优先寻找补充品类、差异化款式或非排他窗口切入

### 2. 补证推进

适用条件：

- 基本面较好
- 但缺少规模、付款方式或合作承诺等关键信息

建议动作：

- 优先补齐采购额、门店数、付款方式
- 获取合作计划书或年度预算
- 验证是否存在竞品代理冲突
- 核验现有供应商合作深度、替换周期和决策门槛

### 3. 培育观察

适用条件：

- 有一定潜力
- 但当前规模或渠道尚不达标

建议动作：

- 维持联系频次
- 观察扩店、跨境布局、增长趋势
- 适合提供轻量样品和市场资料
- 持续跟踪其供应商变化、品类补位机会和新品牌导入窗口

### 4. 低优先级跟进

适用条件：

- 主营偏差明显
- 账期要求强
- 风险高于收益

建议动作：

- 降低销售投入
- 仅保留基础跟踪
- 如有战略原因，再单独报批
