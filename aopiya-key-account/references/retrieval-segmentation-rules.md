# 客户重要性标记规则

本文件用于在已有客户输入基础上完成重要性标记和优先级判断。

## 一、优先标记维度

批量筛客时，优先使用以下判断维度：

1. 联系人、邮箱、电话、网址、地址等联络信息
2. 国家或地区
3. 客户类型
4. 主营品类
5. 年采购额
6. 门店或分销规模
7. 付款方式
8. 决策人经验
9. 当前合作阶段

## 二、建议执行顺序

默认按以下顺序推进：

1. 先确认输入是否具备基本客户主体和联络可达性信息
2. 再按国家、客户类型、主营品类等条件判断行业与客群匹配度
3. 再按规模、付款方式、决策条件完成重要性标记
4. 最后给出优先级、培育价值、补证需求等明确结论

这里的“筛选”本质上是重要性标记，不是剔除客户，而是为候选客户池打上优先级标签，便于销售投入排序。

### 1. 先看联络可达性

重点关注：

- 是否有联系人姓名和职位
- 是否有可用邮箱或电话
- 是否有官网或可验证网址
- 是否有完整地址可辅助定位客户主体

如果联络路径缺失，默认不能直接列为“优先推进”，应至少先补证。

### 2. 再看行业匹配

- 高匹配：箱包
- 中高匹配：鞋品
- 中匹配：女装配饰
- 低匹配：男装及偏离皮具主线的业态

### 3. 再看规模门槛

优先关注：

- 年采购额不低于 1000 万人民币
- 门店或合作分销不少于 15 家
- 有直营网点、加盟网点或跨境布局

### 4. 再看交易条件

优先级从高到低：

1. 现金付款
2. 有担保的 LC
3. 账期客户

### 5. 最后看决策人与合作意愿

重点关注：

- 时尚行业经验是否成熟
- 是否接受长期合作和保底采购
- 决策链是否足够短

## 三、重要性标记标签

完成输入整理和初步判断后，可使用以下标签组合：

### 1. 按客户价值分层

- 战略型大客户
- 重点开发客户
- 储备客户
- 低优先级客户

### 2. 按行业匹配度分层

- 高匹配
- 中匹配
- 低匹配

### 3. 按开发难度分层

- 快速成交型
- 需要培育型
- 高风险型

## 四、批量重要性标记输出要求

批量结果建议至少输出下列列名：

| 列名 | 说明 |
|---|---|
| 公司名称 | 客户主体 |
| 国家/地区 | 市场定位 |
| 客户类型 | 连锁零售、品牌商等 |
| 主营品类 | 用于判断行业匹配 |
| 进入判断依据 | 说明为何进入候选池 |
| 联络信息状态 | 已可触达 / 部分缺失 / 严重缺失 |
| 年采购额 | 用于判断规模 |
| 门店/分销规模 | 用于判断覆盖 |
| 付款方式 | 用于判断资金条件 |
| 是否进入评分 | 信息充分时标记“可进入评分”，否则写“需补关键事实” |
| 客户分层 | 战略、重点、储备、低优先级 |
| 重要性标记 | 优先推进、补证推进、培育观察、低优先级跟进 |
| 标记依据 | 风险或补充点 |

## 五、重要性标记说明模板

不要只写“高优先级”或“低优先级”，应归因到具体条件：

- 联络信息完整，可直接触达
- 联络路径部分缺失，需先补证
- 行业匹配度不足
- 采购规模不足
- 市场覆盖不足
- 付款条件不符合
- 决策链过长
- 合作承诺不足
- 信息缺失，暂无法判断
